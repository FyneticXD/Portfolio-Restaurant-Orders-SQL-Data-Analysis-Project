# Restaurant Orders Analysis - SQL

## 📊 Project Description

This project is a SQL data analysis portfolio using a restaurant orders dataset. The analysis focuses on exploring menu performance, customer ordering behavior, and sales trends to generate business insights that support data-driven decision-making.

## 📂 Dataset

The dataset consists of restaurant menu and customer order data, including:

- Menu Items
- Menu Categories
- Item Prices
- Order Details
- Order Date
- Quantity Ordered

## 💡 Analysis & Key Insights

This SQL project includes the following analyses:

- **Database Exploration**: Understanding the structure of tables and available data.
- **Menu Analysis**: Identifying the most and least expensive menu items, total menu items, and menu distribution by category.
- **Order Analysis**: Analyzing total orders, order frequency, and customer purchasing patterns.
- **Sales Analysis**: Calculating revenue and identifying top-selling menu items.
- **Category Performance**: Comparing sales performance across menu categories.
- **Business Insights**: Producing insights that can support menu evaluation and business decision-making.

## 🛠️ Technologies Used

- **Database**: MySQL
- **Language**: SQL
- **SQL Concepts**:
  - SELECT
  - WHERE
  - ORDER BY
  - GROUP BY
  - JOIN
  - Aggregate Functions (COUNT, SUM, AVG, MIN, MAX)
  - Subqueries

## 🚀 How to Run the Project

1. Clone or download this repository.
2. Import `create_restaurant_db.sql` into MySQL.
3. Execute the SQL scripts in sequence to reproduce the analysis.
4. Review the query results and business insights.

## 📈 Business Value

The analysis helps answer several business questions, such as:

- Which menu categories generate the highest revenue?
- Which menu items are the most and least popular?
- What are customer ordering patterns?
- Which products should be prioritized for marketing or menu optimization?
